## 섹션 1 — PRODUCT.md 초안

```markdown
# Product

## Register
product

## Users
1인 개발자/정보 수집가. 이동 중이나 잠시 여유 시간에 "나중에 봐야지"하는 콘텐츠를
빠르게 저장하고, 자리 잡고 앉아서 다시 찾아보는 사람.
- 저장 시 상태: 손이 바쁘고 맥락 전환이 귀찮다. 던지고 닫고 싶다.
- 조회 시 상태: 여유롭고 집중. "아 그거 뭐였더라" 하는 필터/정렬로 30초 안에 찾는다.
  (MVP는 검색 없음, 필터+정렬 중심)

## Product Purpose
흩어지는 정보(영상·URL·캡처·메모)를 형태·상태·날짜 3축으로 분류하고,
필터/정렬만으로 나중에 다시 찾을 수 있도록 보관하는 개인 도구. (MVP 범위: 검색 없음)
성공 기준: 저장한 항목을 1주일 뒤에도 30초 안에 찾을 수 있다.

## Brand Personality
따뜻한 · 개인적인 · 군더더기없는

## Anti-references
- Notion: 문서 중심 계층 구조, 높은 진입 마찰
- 다크 SaaS 대시보드: 파란 그라디언트, 글로우 효과, AI 느낌 과시
- 순백색 배경 + 회색 카드 (Bootstrap/Material 기본 느낌)
- 뉴스/피드 앱: 썸네일 그리드, 정보 과부하, 소비 유도 디자인

## Design Principles
1. 저장은 반사적으로, 조회는 집중적으로 — 두 모드의 마찰을 따로 최적화한다
2. 도구가 배경으로 사라진다 — UI가 콘텐츠보다 눈에 띄면 실패
3. 따뜻한 비어있음 — 빈 상태조차 "내 공간"처럼 느껴져야 한다
4. 3축으로 끝낸다 — 태그·폴더·계층은 추가하지 않는다

## Accessibility & Inclusion
- WCAG AA 준수
- 모바일 터치 타깃 최소 44×44px
- 시스템 다크 모드 지원 (기본: 라이트)
- 한국어 전용 UI
```

## 섹션 2 — DESIGN.md Seed 초안

### 테마 씬 문장
> "저녁 7시, 소파에 기대어 폰을 꺼내 오늘 저장한 것들을 훑는다. 방에는 전구색 조명이 켜져 있다."

라이트 모드, 따뜻한 off-white 베이스. 형광등 같은 순백색은 없다.

### 컬러 전략
Restrained (product 기본) — 액센트 1색, 따뜻한 amber/terracotta

> **주의**: border chroma는 0.006~0.008 사이 유지 (높이면 "누런" 느낌).
> text-primary는 대비를 충분히 (라이트에서 oklch 22% 이하).
> 상태 구분은 색보다 문구/위치/아이콘으로. 썸네일 있는 항목에서 색채가 자연스럽게 들어온다.

```css
/* Light mode */
--bg:           oklch(97% 0.006 80);   /* 따뜻한 off-white */
--surface:      oklch(99% 0.003 80);   /* 카드/패널 */
--border:       oklch(87% 0.007 80);   /* 따뜻하되 누렇지 않게 */
--text-primary: oklch(18% 0.008 80);   /* 충분히 짙게 — 대비 우선 */
--text-sub:     oklch(52% 0.010 80);
--accent:       oklch(62% 0.14  55);   /* terracotta/amber 액센트 */
--accent-bg:    oklch(93% 0.04  55);   /* 선택된 상태 배경 */

/* Dark mode */
--bg:           oklch(16% 0.010 80);
--surface:      oklch(21% 0.008 80);
--border:       oklch(32% 0.010 80);
--text-primary: oklch(90% 0.006 80);
--text-sub:     oklch(65% 0.008 80);
--accent:       oklch(68% 0.13  55);
--accent-bg:    oklch(28% 0.06  55);
```

### 타이포그래피
- 폰트: `"Pretendard", -apple-system, "Noto Sans KR", sans-serif`
- 스케일: 12 / 14 / 16 / 20px (product register — tight)
- 웨이트 대비: 400(본문) / 500(레이블) / 600(제목)
- 줄 길이: 65ch 이하 (메모/제목 텍스트)

### 간격
- 베이스 단위: 4px
- 주요 값: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 48px

### 컴포넌트
- border-radius: 리스트 아이템 6px / 버튼 8px / 입력 6px / 칩 999px
  > 10px 이상은 앱스토어 카드 느낌 — 이 앱은 리스트형 도구이므로 납작하게
- 그림자: `0 1px 3px oklch(20% 0.01 80 / 0.08)` (라이트), 없음(다크)
- 버튼: primary = accent 배경, secondary = surface + border
- 뱃지/칩: accent-bg + accent 텍스트

### 모션
- 전환: 150~200ms, `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-quint)
- 상태 전달 목적만 — 데코레이션 없음

## 섹션 3 — 메인 목록 화면 Shape Brief

**Feature Summary**
저장된 항목의 목록을 보여주는 홈 화면. 형태/상태 필터 + 목록 + FAB으로 구성.
사용자의 주요 작업은 "오늘 저장한 것 찾기"와 "상태 변경(안 봤음→봤음)"이다.

**Primary User Action**
필터를 눌러 범위를 좁힌 뒤, 원하는 항목을 찾아 탭한다.

**Design Direction**
- 컬러 전략: Restrained — accent는 현재 선택된 필터/상태에만
- 레퍼런스: Bear(따뜻함) + Linear(밀도와 기능성)

**Layout Strategy**
```
┌─────────────────────────────┐
│  [로고]          [설정 아이콘] │  ← sticky header (얇게)
├─────────────────────────────┤
│  [전체][영상][글][캡처][메모]  │  ← 형태 칩 (horizontal scroll)
│  [안봤음] [봤음] [보관]       │  ← 상태 탭
├─────────────────────────────┤
│                             │
│  제목 텍스트 (한 줄)           │
│    메모 미리보기 (최대 2줄)    │
│    날짜 · 형태뱃지            │  ← 리스트 행 (카드 그리드 아님)
│  ──────────────────────    │
│  제목 텍스트                  │
│    ...                      │
│                             │
│                       [+]   │  ← FAB (우하단)
└─────────────────────────────┘
```

**Key States**
| 상태 | 표시 내용 |
|---|---|
| 기본 목록 | 항목 리스트, 날짜 그룹 구분선 (오늘/이번 주/이전) |
| 빈 상태 (최초) | "+ 버튼으로 첫 항목을 추가해보세요" + 사용 방법 한 줄 |
| 빈 상태 (필터) | "이 분류에는 아직 없어요." + 필터 초기화 링크 |
| 로딩 | skeleton 행 3개 (spinner 없음) |
| 이미지 있는 항목 | 우측에 썸네일 48×48 |

**Interaction Model**
- 항목 탭 → 상세 화면
- 좌스와이프 → 즉시 "봤음" 처리 (빠른 상태 전환)
- 길게 누르기 → 바텀 시트 (안봤음 / 봤음 / 보관 선택)
- 데스크톱 hover → 항목 우측에 작은 상태 버튼 노출
- 형태 칩: 단일 선택, 탭하면 토글
- 상태 탭: 단일 선택
- FAB → `/new` 화면
- 무한 스크롤 (초기 MVP)

## 섹션 3b — 저장 화면 Shape Brief (새 항목 추가 / Web Share 진입)

> **MVP의 핵심 승부처**: 외부 앱 공유 → 5초 안에 저장 완료

**Feature Summary**
외부 앱(유튜브, 브라우저 등) 공유 버튼 또는 FAB으로 진입한 새 항목 저장 화면.
Web Share Target으로 진입 시 URL/이미지가 미리 채워진다.
사용자는 자동으로 채워진 내용을 확인하고 거의 수정 없이 저장한다.

**Primary User Action**
자동 추정된 형태를 확인하고 `저장하기` 버튼을 누른다. 입력은 선택사항.

**Design Direction**
입력 필드 최소화 — 보여주는 화면이지 입력 화면이 아니다.

**형태 자동 추정 규칙**
| 조건 | 추정 형태 |
|---|---|
| `youtube.com` / `youtu.be` 도메인 URL | 영상 |
| 그 외 URL | 글 |
| URL 없고 이미지만 있음 | 캡처 |
| 둘 다 없음 | 메모 |

> 칩으로 즉시 변경 가능 — 자동 추정은 제안일 뿐

**Layout Strategy**
```
┌─────────────────────────────┐
│  ← 닫기              저장   │  ← 상단 바 (최소화)
├─────────────────────────────┤
│  ┌──────────────────────┐   │
│  │ 썸네일 / 이미지 미리보기│   │  ← URL이면 og:image, 이미지면 미리보기
│  └──────────────────────┘   │
│                             │
│  제목 (자동추출, 편집 가능)   │
│  메모 입력 (optional)        │
│                             │
│  형태: [영상][글][캡처][메모] │  ← 자동 선택됨, 탭으로 변경
│  상태: 안봤음                │  ← 작은 칩으로 표시, 탭하면 바텀 시트
├─────────────────────────────┤
│         [ 저장하기 ]          │  ← accent 버튼, 풀 너비
└─────────────────────────────┘
```

**Key States**
| 진입 방식 | 채워진 내용 |
|---|---|
| Web Share (URL) | URL + og:title + og:image 자동 입력, 형태 자동 추정 |
| Web Share (이미지) | 이미지 미리보기, 형태 = 캡처 |
| FAB 진입 | 빈 상태, 형태 = 메모 기본 |

**저장 성공 후 피드백**
- 체크 아이콘 + "저장됐어요" 토스트 1.5초
- Web Share 진입: 앱 닫기 or 목록 복귀 선택
- FAB 진입: 목록 화면으로 자동 복귀

## 섹션 4 — 추천 디자인 Lane

**Lane 이름**: Warm-Minimal Personal Tool

### 정의
| 항목 | 결정 |
|---|---|
| 배경 | 따뜻한 off-white — 형광등 순백색 없음 |
| 구조 | 리스트 우선, 밀도 있는 행, 카드 그리드 없음 |
| 타입 | Pretendard — 한국어 최적화, 따뜻한 가독성 |
| 액센트 | terracotta/amber 단일 컬러 — 선택 상태, primary 버튼만 |
| 여백 | 필터 영역과 목록 영역 간격 차이를 리듬으로 활용 |
| 감성 | "내 물건이 모여있는 서랍" — 도구적이면서도 개인적 |

### 기준 레퍼런스 조합
- **Bear (iOS)**: 따뜻한 tone, 목록 중심, 간결함
- **Linear**: 밀도 있는 행, 빠른 필터
- **Things 3**: 상태 관리 명확함, 리스트 뷰의 가독성

### 선택 이유
- "따뜻하고 개인적인" + 도구적 기능성의 교집합
- 카드 그리드(뉴스 앱), 다크 SaaS(파란 그라디언트), 문서 툴(Notion) 세 안티레퍼런스를 모두 피함
- 모바일 PWA에서 리스트 뷰가 카드 그리드보다 스크롤 효율적

## 섹션 5 — 피해야 할 AI Slop 패턴

### 공통 절대 금지
| 패턴 | 대안 |
|---|---|
| 사이드 보더 accent (`border-left` 3px+ 컬러) | 배경 tint 또는 없앰 |
| 그라디언트 텍스트 (`background-clip: text`) | 단색 액센트 컬러 |
| 동일 크기 카드 반복 그리드 | 텍스트 중심 리스트 행 |
| 모달을 첫 번째 해결책으로 | 인라인 액션 or 바텀 시트 |

### 이 프로젝트 특화 패턴
| 패턴 | 왜 안 되는가 | 대안 |
|---|---|---|
| 뉴스/피드 썸네일 그리드 | 안티레퍼런스 직접 해당 | 텍스트 우선 리스트, 썸네일은 보조 |
| 순백색 배경 + 회색 카드 | 안티레퍼런스 직접 해당 | 따뜻한 off-white 베이스 |
| 파란 계열 액센트 (blue/indigo/violet) | 차가운 SaaS 연상 | amber/terracotta 유지 |
| 히어로 메트릭 ("저장한 항목 47개") | 개인 도구에 불필요한 대시보드 느낌 | 없앰 |
| "Nothing here yet" 빈 상태 | 빈 공간을 낭비로 처리 | 따뜻하게 유도하는 문구 + 행동 유도 |
| 글로우 / 블러 / glassmorphism | 다크 SaaS 연상 | 없앰 |
| 다크 모드를 기본으로 설정 | 씬 문장이 라이트를 강제 | 라이트 기본, 시스템 따라가기 |
| FAB에 그라디언트 | AI 앱 클리셰 | accent 단색 |
| 상태 변경 전용 모달 | 모달 남용 | 좌스와이프(즉시) or 길게 누르기(바텀 시트) |
| border-radius 10px 이상 카드 | 앱스토어 카드 느낌 | 6px 이하 유지 |
