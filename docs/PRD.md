# PRD — Check Later

## 목표

저장 마찰을 카톡 나에게 보내기 수준으로 낮추면서, 카톡에 없는 분류·조회를 동시에 제공한다.

- Web Share Target API로 외부 앱 "공유" 메뉴에서 바로 저장
- 3축 분류(형태 / 상태 / 날짜)만으로 끝냄 — 자유 태그 없음
- 모바일/PC 어디서나 같은 데이터 (Supabase 클라우드 동기화)
- 개인 도구이지만 다중 사용자 구조 (Supabase Auth + RLS)

## 사용자

1인 개발자/정보 수집가. 이동 중이나 잠시 여유 시간에 "나중에 봐야지"하는 콘텐츠를 빠르게 저장하고, 자리 잡고 앉아서 다시 찾아보는 사람.

- **저장 시**: 손이 바쁘고 맥락 전환이 귀찮다. 던지고 닫고 싶다.
- **조회 시**: 여유롭고 집중. 필터/정렬로 30초 안에 찾는다.

성공 기준: 저장한 항목을 1주일 뒤에도 30초 안에 찾을 수 있다.

## 핵심 기능 (MVP 포함)

- 회원가입/로그인 (Google OAuth)
- 항목 저장: 텍스트 메모 / URL / 이미지(캡처) 중 하나 이상
- 분류: 형태(영상·글·캡처·메모) + 상태(안 봤음·봤음·보관) + 날짜(최근순)
- 목록 조회 + 필터(`type`, `status`) + 정렬(`created_at desc` 고정)
- 상태 변경 (안 봤음 → 봤음 → 보관)
- PWA 홈화면 설치 + Web Share Target

## MVP 제외

| 제외 | 이유 |
|---|---|
| 능동 알림/리마인더 | 사용자 합의 — 알림 없이 수동 조회만 |
| 자유 태그 / AI 자동 분류 | 입력 마찰 증가, 결국 안 쓰게 됨 |
| 디스코드/카카오 봇 | V2 후보. PWA가 어차피 필요하므로 봇은 추가 부담 |
| 사용자 간 공유 | 개인 도구 정체성 유지 |
| 검색 | 필터+정렬 중심 MVP. V2 후보 |
| 이메일 로그인 | Google OAuth로 통일 — Supabase 인증 관리 단순화 |
| 날짜 범위 필터 | MVP는 `created_at desc` 고정 정렬만. V2 후보 |

## Web Share Target 동작

외부 앱 "공유" 메뉴에서 `/new?title=&text=&url=` 형태로 진입한다.

| 파라미터 | 저장 필드 | 비고 |
|---|---|---|
| `title` | `title` | 비어 있으면 `url` 또는 사용자 입력으로 대체 |
| `text` | `memo` | 짧은 메모로 저장 |
| `url` | `url` | URL이면 type 자동 판정 |
| `files` | — | MVP 제외. 이미지 공유 수신은 V2 후보. MVP에서는 앱 내부 `/new`에서 직접 업로드만 지원 |

**manifest 설정**: `method: GET`, 파라미터는 `title`, `text`, `url` 3개만.

**진입 후 동작**: 자동 저장이 아닌 확인 화면(`/new`)으로 이동한다. 사용자가 type·memo를 확인·수정 후 저장 버튼을 누른다. 빠른 저장이 필요하면 기본값 그대로 저장 가능.
